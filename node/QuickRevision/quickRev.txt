🔹 What is a Route?

In Express.js, a route is:

A rule that tells your server what to do when a user visits a specific URL or in simple
a route is nothing but to prepare what to reply to the server when user enters a specific url
🔹 Types of Routes
1. Static Route
app.get('/about', ...)
Fixed path
Always same
2. Dynamic Route
app.get('/blog/:slug', ...)
:slug = variable
Can change based on URL

👉 /blog/react
👉 /blog/node

Same route, different data.

🔹 What is req and res?

Every route has this:

(req, res)
req → what user sends (URL, data, params)
res → what server sends back

get request is used to get something from the server it maybe page or function etc
post request is used to give something to the server like passwords etc .
🧠 Proper understanding:
Node.js → runtime (runs JS outside browser)
Express.js → framework to handle routes
Server → the program you build using Node + Express
👉 So:
Server = your backend app running on Node using Express
